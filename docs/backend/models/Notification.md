# Notification Model – Documentation

## Overview

Defines the **Notification** Mongoose model for user, system, and status updates in CFH.

## Fields

- **userId:** Reference to User (indexed)
- **type:** Enum – System, Message, DisputeUpdate, LoanUpdate, InspectionUpdate, DeliveryUpdate, AuctionActivity, PaymentStatus, BadgeEarned
- **title:** Message title (max 200 chars)
- **message:** Main content (sanitized)
- **link:** (optional) Link to resource or detail
- **isRead:** Read status (indexed)
- **createdAt/expiresAt:** Timestamps for display and expiry

## Indexes

- `(userId, isRead)` – Quick lookup for unread notifications

## Methods

- `markAsRead()`: Sets isRead to true
- **Statics:** `expireOld()` – Removes notifications past expiresAt

## Tier-based Features

- **Premium/Wow++:** Notification digests, real-time in-app popups, advanced filtering/search
- **Free/Standard:** Basic notifications, manual read/mark/delete
