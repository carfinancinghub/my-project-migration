# InsurancePolicy – Documentation

## Overview

Defines the **InsurancePolicy** Mongoose model for insurance policy management in CFH, with risk scoring, checklist validation, audit logging, and tier-aware features.

## Main Features

- **Policy details:** policyId, vehicleId, quote amount, status (draft/pending/approved), conditions
- **Checklist:** Array of checklist items for validation
- **Risk Score:** Calculated on save (placeholder for AI scoring)
- **Audit Log:** Array of action/user/timestamp records, static search by action/user
- **Virtual field:** Policy duration (hours between createdAt/updatedAt)
- **Compound indexes:** For query speed (policyId+status, vehicleId+premium)
- **Pre-save hooks:** Checklist validation, auto risk scoring

## Tier-based Extensions

- **Premium:** Advanced risk score calculation, AI integration, longer audit log retention
- **Free:** Core policy and checklist features

## Methods & Statics

- `logAudit(action, user)` – Pushes to audit log
- `calculateRiskScore()` – Returns risk score (placeholder logic)
- `searchAudit(action, user)` – Static for querying audit logs
